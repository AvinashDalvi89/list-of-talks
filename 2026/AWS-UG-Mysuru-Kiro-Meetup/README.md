
# Beyond the Prompt: Spec-Driven Development with Kiro and the AI-Driven Development Lifecycle

**AWS UG Mysuru April 2026 Meetup - Kiro**

> *"Vibe coding got us here. Spec-driven development takes us further."*

## Talk Summary

Most developers are using AI to write code. Far fewer have a process around it. This talk covers the shift from prompt-driven experimentation to spec-driven development — and how Kiro and AIDLC give that shift a structure you can actually use.


## What this talk covers

- Why vibe coding breaks at team and enterprise scale — from real production experience
- The two anti-patterns AWS identified: AI-Assisted and AI-Autonomous, and why both fall short
- AI-Driven Development Lifecycle (AIDLC) — AWS's reimagined methodology built for the AI era
- Kiro as the execution engine — Spec mode, Steering files, and AIDLC Powers
- Real stories: SMS vendor crisis, the bug that kept returning, the three workflows that were one
- Brownfield reality — how AIDLC handles existing codebases, not just greenfield demos
- How to start this week with open-source steering files
